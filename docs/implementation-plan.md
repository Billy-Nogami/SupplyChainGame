# План реализации

## Этап 1. Базовый каркас

- инициализировать Go-модуль;
- зафиксировать слои Clean Architecture;
- поднять минимальный HTTP API;
- реализовать комнату и подключение игроков.

## Этап 2. Домен игры

- описать сущности `Scenario`, `GameSession`, `NodeState`, `WeekState`, `Pipeline`;
- определить роли, статусы и инварианты;
- формализовать шаг одной недели.

## Этап 3. Use cases

- создание комнаты;
- подключение игрока;
- назначение ролей;
- старт игры;
- отправка заказа;
- переход к следующей неделе;
- получение состояния и истории.

## Этап 4. Хранение данных

- in-memory хранилище для MVP;
- Redis для активных игровых сессий и real-time комнат;
- PostgreSQL для сценариев, архивов игр и аналитики.

## Этап 5. Реальное взаимодействие

- WebSocket или SSE для обновления состояния комнаты;
- синхронизация ходов игроков;
- таймауты и auto-policy для пропущенных ходов.

## Этап 6. Аналитика и экспорт

- расчет total cost, backlog, average inventory;
- расчет bullwhip effect;
- выгрузка истории сессии в `.xlsx`.

## Этап 7. Симуляции

- автоматические стратегии;
- batch simulation;
- Monte Carlo режим.
